# Support

AirBridge is an early local-network transfer tool. The fastest way to get useful help is to include the network and platform details that affect discovery and file transfer.

## Before Opening an Issue

Check these first:

1. Both devices are on the same Wi-Fi or reachable LAN.
2. Windows Firewall allows AirBridge on private networks.
3. The receiving app is running and shows a local address.
4. Manual peer entry works with the address shown by the other device.
5. The file is below the app's documented size limits.

## Where to Ask

- Use a bug report issue for crashes, discovery failures, transfer failures, update problems, or packaging issues.
- Use a feature request issue for new platform support, protocol changes, UI improvements, or release workflow ideas.
- Use the security policy for vulnerabilities, unsafe file handling, or update-safety concerns.

## Useful Details

When asking for help, include:

- Sender and receiver platforms.
- AirBridge version or release asset.
- Whether automatic discovery or manual peer entry was used.
- Local addresses shown by both devices, with any private details removed if needed.
- Firewall prompts, VPNs, school networks, office networks, or router restrictions.
- A short description of the file or message being sent.

## Scope

AirBridge is designed for trusted local networks. It does not provide a cloud relay, account system, public-network hardening, or end-to-end encryption.
